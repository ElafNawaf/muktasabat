6Multi-Tenancy Architecture — PostgreSQL Row-Level Security (RLS)

PropOS uses PostgreSQL Row-Level Security (RLS) for tenant isolation. A single shared database contains all tenant data, but Postgres RLS policies ensure each tenant only queries their own rows. The tenant_id from the JWT is injected into every DB session via SET LOCAL app.tenant_id = '...' and RLS policies filter automatically with zero application-layer overhead.

🔑 Request Flow

  • Client sends request with JWT Bearer token
  • Auth service validates JWT, extracts tenant_id claim
  • NestJS DB middleware: SET LOCAL app.tenant_id
  • PostgreSQL RLS policies filter all queries automatically
  • No cross-tenant data leakage possible at DB level
  • PgBouncer pools connections in transaction mode

🏗️ Schema Standards

  • tenant_id UUID on every entity table (NOT NULL)
  • POLICY: USING (tenant_id = current_setting(...)::uuid)
  • Composite indexes: (tenant_id, status, created_at DESC)
  • Soft deletes: deleted_at TIMESTAMPTZ DEFAULT NULL
  • Audit: created_at, updated_at, created_by, updated_by
  • Schema-per-tenant option for large enterprise clients

🔓 Super Admin Access

  • Privileged DB role bypasses RLS for platform ops
  • Used exclusively by super-admin service pod
  • All cross-tenant access recorded in audit_log
  • Separate read replica for platform-wide analytics
  • Feature flag management per tenant via settings JSONB
  • Tenant suspension: UPDATE tenants SET status='suspended'

9Security Architecture Summary

🔐 Authentication & Identity

  • OAuth2 + JWT (15-min access / 7-day refresh)
  • MFA mandatory for Finance & Executive roles
  • TOTP (Authenticator app) + SMS fallback
  • SSO via SAML 2.0 for enterprise clients
  • Biometric login on iOS/Android apps
  • JWT stored in httpOnly Secure cookies (web)

🛡️ Data Protection

  • AES-256 encryption at rest (RDS + S3)
  • TLS 1.3 for all traffic in transit
  • Field-level encryption: National ID, MFA secrets
  • Data residency: AWS Riyadh (me-south-1)
  • PDPL compliance (Saudi data protection law)
  • NCA Essential Cybersecurity Controls

📋 Audit & Compliance

  • Immutable audit log — every DB mutation logged
  • Before/after JSON for all UPDATE operations
  • ZATCA Phase 2 e-invoicing (Fatoorah API)
  • Annual external penetration testing (CREST)
  • ISO 27001 certification target (Year 2)
  • SOC 2 Type II certification target (Year 2)
